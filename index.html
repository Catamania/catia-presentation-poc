Titre

  • https://angular.io/guide/architecture
  • https://angular.io/guide/glossary
  • Une Single-page application est une application qui s'éxecute dans le navigateur. Et qui ne requiert pas de recharger la page pendant l'utilisation.
  • Un module Angular est un conteneur logique qui regroupe des composants et des services (et ...). Le module déclare ses composants et ses services
    (metadata
    https://alligator.io/angular/anatomy-angular-module/
    @NgModule metadata tells the Angular compiler what components to compile for this module and how to link this module with other modules.
    https://angular.io/guide/ngmodule-faq#q-angular-compiler)
  • Une vue est une portion de l'écran qui affiche de l'information et écoute les actions utilisateur (clic, souris, clavier)
  • Un composant est une classe chargée d'afficher des données dans une vue, de gérer les interactions utilisateur pour cette vue. Un composant Angular est une brique élémentaire du front. Une application angular est un arbre de composants. (Ou un arbre de vues) Cf @NgModule "bootstrap: [AppComponent]"
    (metadata
    https://angular.io/api/core/Component)
  • Un service Angular : implémentation de logique transverse. Un service est partagé, asychrone, injectable
    (metadata
    https://angular.io/guide/architecture#services-and-dependency-injection)
  • React : on écrit du html dans du js. Angular : du js dans du html (markup)
